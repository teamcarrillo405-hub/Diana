# Subject Methodology Audit (P9-A, 2026-06-10)

Every high-school subject helper audited against three criteria:
**(1) Discipline method** — does the scaffold match the subject's
evidence-based way of thinking? **(2) Memory** — does work feed spaced
repetition / concept mastery? **(3) Organization** — does it produce steps,
checklists, or structure the student can hold onto?

| Subject | Discipline method in the scaffold | Memory | Organization | Verdict |
|---|---|---|---|---|
| Math | Socratic hints (never the answer), worked examples on *analogous* problems (worked-example effect), photo→step board, unit tracker | Cards from assignments; concept mastery | Step board + **digit-grid workspace (new)** — column alignment structurally enforced for dysgraphia | Sound |
| Science | Hypothesis-first; lab-report scaffold ≈ claim-evidence-reasoning; chemistry balancing by atom-counting iteration (never the balanced answer) | Cards; concepts | Scientific-method step modes | Sound; **follow-up: a dimensional-analysis unit rail** (fraction-style unit cancellation grid, sibling of the math grid) |
| History | Sourcing/corroboration (primary-source mode), cause-effect, HAPP, DBQ six-part outline — matches the historical-thinking canon (SHEG) | Cards; concepts | DBQ outline contract | Sound |
| English / Writing | Structure help and one-rule explanations, never drafting; ghost text only on explicit accept; evidence finder pulls *the student's own notes* | Vocabulary → FSRS | Outline/paragraph structure modes | Sound |
| CS | Error-hint-first (guiding question before any code), pseudocode bridge, real CPython sandbox | Cards | Debug-log scaffold | Sound |
| Language | Comprehensible-input leaning: cognate hints, interest-based sentences, target-language STT speaking practice | Vocab cards with TTS both sides | Conjugation/vocab modes | Sound |
| Arts | Reflection-first (process before product), formal analysis for art history, deterministic music theory | Cards (theory) | Storyboard scaffold | Sound |
| Health / PE | Factual, age-appropriate, no body metrics by design; CPR/first-aid → FSRS | CPR cards | Movement goals | Sound |
| AP (all) | Format-true FRQ outlines per exam, MCQ with explanations for *every* choice, banded (never pass/fail) self-estimates | Practice attempts feed mastery | Day-based milestone plans + test-prep engine | Sound |

## Cross-cutting strengths (the methodology spine)
Capture → one move → scaffold-never-substitute → rubric self-check →
grade-informed recovery → spaced memory → effectiveness loop adapting which
shape of help each student gets. Memory and organization are not per-subject
afterthoughts — they are the shared substrate every helper feeds.

## Gaps found and disposition
1. **Math column alignment (dysgraphia)** — closed this session: digit-grid
   workspace (one digit per cell, carry row, division bring-down discipline).
2. **Science unit alignment** — open follow-up: a unit-cancellation rail
   (fraction grid where units visibly cancel) using the same grid substrate.
3. **Pen-input ink realignment** — out of scope for the web app; requires
   handwriting recognition; revisit with the native (Capacitor) build.

No methodology defects found that contradict the discipline canons; the
helpers were built against them phase by phase and the audit confirms it.
