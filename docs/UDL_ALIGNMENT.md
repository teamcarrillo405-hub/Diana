# Diana × Universal Design for Learning (UDL)

Mapping of Diana features to the CAST UDL Guidelines (udlguidelines.cast.org).
Written for IEP teams, special-education coordinators, and district reviewers:
every accommodation in Diana is an implementation of a published, evidence-based
UDL checkpoint — not an invented intervention.

## Principle 1 — Multiple Means of Engagement (the "why")

| UDL checkpoint | Diana implementation |
|---|---|
| 7.1 Optimize individual choice and autonomy | Energy modes (Low / OK / On it), accent + theme personalization, student-controlled AI traffic light per class and assignment |
| 7.2 Optimize relevance, value, authenticity | Interest-based analogies in every subject helper (student picks up to five interests at onboarding) |
| 7.3 Minimize threats and distractions | Calm invariant: no red error states, no streaks, no shame language (enforced by an automated tone audit); reduced-motion support |
| 8.2 Vary demands and resources to optimize challenge | "Next 5 minutes" scoring adapts to energy, mood, sleep recovery, and rough-mode |
| 8.3 Foster collaboration and community | Invite-only study groups: shared decks, notes, sessions — no leaderboards or rankings |
| 9.1 Promote expectations and beliefs that optimize motivation | First-week guided journey; quiet wins (private, text-only milestones) |
| 9.2 Facilitate personal coping skills and strategies | "I'm overwhelmed" button → smallest next step; burnout cue; weekly reflection |
| 9.3 Develop self-assessment and reflection | Rubric self-check (private, student-driven); mood check-in; authorship ownership meter |

## Principle 2 — Multiple Means of Representation (the "what")

| UDL checkpoint | Diana implementation |
|---|---|
| 1.1 Offer ways of customizing the display | Font size, line spacing, letter/word spacing, dyslexia-friendly fonts (Lexend, Atkinson Hyperlegible, OpenDyslexic), high contrast, dark/light/system themes |
| 1.2 Offer alternatives for auditory information | Lecture transcription (speech-to-text) with AI cleanup and outlines |
| 1.3 Offer alternatives for visual information | Text-to-speech with word highlighting across reading surfaces; multiple TTS providers and speeds |
| 2.1 Clarify vocabulary and symbols | Vocabulary hover support: context clues, phonics, on-demand definitions; saved to spaced-repetition cards |
| 2.2 Clarify syntax and structure | Reading-level adaptation (simpler / more detail); bionic reading; line focus |
| 2.5 Illustrate through multiple media | Visual learning tools: mind maps, concept graphs, timelines, comparison tables, math step boards |
| 3.1 Activate or supply background knowledge | Pre-reading scaffolds; gap-bridge suggestions linking weak concepts to strong ones |
| 3.4 Maximize transfer and generalization | FSRS-5 spaced repetition (published Anki v5 algorithm) seeded from the student's own notes |

## Principle 3 — Multiple Means of Action & Expression (the "how")

| UDL checkpoint | Diana implementation |
|---|---|
| 4.1 Vary the methods for response and navigation | Voice capture everywhere (speech-to-text), photo/document upload, share-sheet capture, keyboard-reachable pacing controls |
| 5.2 Use multiple tools for construction and composition | Writing studio with structure help (never ghost-writing); citation formatting; math photo scan + step board |
| 5.3 Build fluencies with graduated levels of support | Hint ladders (smallest clue first); Socratic-only AI help; worked examples on analogous problems, never the student's own |
| 6.1 Guide appropriate goal-setting | Evening planning; intention setting; AP exam plans with advisory milestones |
| 6.2 Support planning and strategy development | Five-minute task breakdown; templates; focus plans |
| 6.3 Facilitate managing information and resources | Class-routed notes; full-text search; source-anchored study artifacts |
| 6.4 Enhance capacity for monitoring progress | Rubric self-check with points-weighted coverage; grade insights with recovery moves; time-estimate calibration |

## Accommodation honesty

Diana's IEP/504 import maps documented accommodations (extended time, reader,
scribe, breaks, reduced quantity) onto concrete app behaviors (time-estimate
multipliers, TTS defaults, dyslexia typography, session pacing). The dyslexia
reading-time multiplier (1.6×) is applied consistently in both prioritization
and nightly workload budgeting.

## Authorship integrity (for academic-integrity review)

- All AI help is Socratic: server-enforced refusal of final-work requests.
- Every AI interaction is logged to a student-exportable authorship log.
- Teachers can be shown per-assignment AI authorship receipts — feature-level
  usage counts, no content — at the student's discretion.
- Class- and assignment-level AI policy (green/yellow/red) is enforced
  server-side in every AI function, not just in the UI.
