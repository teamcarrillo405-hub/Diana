# Plan 08-01: Provider Wiring — Whisper STT + OpenAI TTS (F4/F6/F8)

## Objective

Wire two new Supabase Edge Functions — `transcribe-voice` (Whisper STT) and `tts-generate` (OpenAI TTS) — and add a `profiles.tts_provider` setting (`'browser' | 'openai'`) so the existing `VoiceTextarea`, `TtsButton`, and `TtsHighlightButton` components opt-in to the higher-quality server path when the user has selected `'openai'`. OCR is intentionally untouched — `classify-inbox` already uses Claude vision per Phase 3.

## Context

The browser Web Speech API (Chromium + Safari only) and `window.speechSynthesis` (varying voice quality across browsers) currently power F4 voice capture, F6 read-aloud, and F8 note dictation. Users on Firefox have no mic; users with English-as-a-second-language often need higher-accuracy STT. Whisper and OpenAI TTS are the user-locked providers (per phase description). All AI calls must go through Edge Functions per CLAUDE.md.

## must_haves
- `supabase/functions/transcribe-voice/index.ts` exists, downloads audio from Supabase Storage, POSTs to OpenAI Whisper, and returns `{ ok: true, text: "<transcript>" }`
- `supabase/functions/tts-generate/index.ts` exists, POSTs to OpenAI TTS, and returns binary `audio/mpeg` with the same CORS headers used by other Edge Functions
- `profiles.tts_provider` column exists (TEXT NOT NULL DEFAULT 'browser', check constraint `('browser','openai')`)
- `VoiceTextarea` accepts a `provider: 'browser' | 'openai'` prop; when `'openai'`, after the user stops dictation, the recorded audio (via MediaRecorder) is uploaded to a temp Storage path and `transcribe-voice` is invoked; the returned text is appended via `onTranscript`
- `TtsButton` and `TtsHighlightButton` accept a `provider: 'browser' | 'openai'` prop; when `'openai'`, they invoke `tts-generate` and play the returned audio via an `<audio>` element instead of `speechSynthesis.speak`
- `lib/profile.ts` returns `tts_provider` on the profile object so callers can read it
- The browser fallback path remains the default and continues to work without code changes

<action>
Create file `supabase/functions/transcribe-voice/index.ts`. The function:

1. Imports `createClient` from `https://esm.sh/@supabase/supabase-js@2`.
2. Reads env vars: `OPENAI_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`.
3. Handles `OPTIONS` preflight with `Access-Control-Allow-Origin: *` and `Access-Control-Allow-Headers: authorization, content-type`.
4. Accepts POST JSON body `{ audioStorageKey: string, bucket?: "note-audio" | "voice-temp" }` (defaults bucket to `"note-audio"`).
5. Returns `{ error: "audioStorageKey required" }` with status 400 if missing.
6. Downloads the blob via `supabase.storage.from(bucket).download(audioStorageKey)`. If `error || !blob` return `{ error: "Audio not found" }` status 404.
7. Builds `FormData`: `formData.append("file", new File([await blob.arrayBuffer()], "audio.webm", { type: blob.type || "audio/webm" }))` (Pitfall 1: never let `type` be empty). Then `formData.append("model", "whisper-1")`.
8. POSTs to `https://api.openai.com/v1/audio/transcriptions` with header `Authorization: Bearer ${OPENAI_API_KEY}` and the FormData body. No `Content-Type` header — fetch sets the multipart boundary automatically.
9. If `!openaiRes.ok` log the response text via `console.error("openai whisper error:", await openaiRes.text())` and return `{ error: "Transcription failed" }` status 500.
10. Parses JSON `{ text } = await openaiRes.json()`.
11. Returns `new Response(JSON.stringify({ ok: true, text }), { status: 200, headers: { "Content-Type": "application/json", "Access-Control-Allow-Origin": "*" } })`.
12. Wraps everything in a top-level `try/catch` that returns `{ error: "Internal error" }` status 500 on any thrown exception. Use the exact CORS + try/catch structure from `transcribe-note/index.ts` lines 38-46 and 166-172.

ALSO modify two files to add `"stt_transcribe"` to the LogParams.feature union:
- `supabase/functions/_shared/safety.ts` line 22: change the union to `"math_step" | "writing_aid" | "citation_gen" | "reading_scaffold" | "transcribe_note" | "stt_transcribe" | "tts_generate"`.
- `lib/ai/safety.ts` line 13-17: apply the identical union update. Both files must remain mirror-synced.

NOTE: This task does NOT add `OPENAI_API_KEY` to the Supabase secrets — that is a manual step that the user performs before deploying. Add a comment at the top of `transcribe-voice/index.ts`: `// REQUIRES: OPENAI_API_KEY in Supabase Edge Function secrets. Run: supabase secrets set OPENAI_API_KEY=sk-...`.

NOTE: Do NOT call `logInteraction` for this function in Wave 1 — it's a non-Claude AI call. Logging may be added in a follow-up if the audit log needs it, but Whisper usage is auditable via OpenAI dashboard.

<action>
Create file `supabase/functions/tts-generate/index.ts`. The function:

1. Reads env var `OPENAI_API_KEY` only (no Supabase client needed — this function doesn't touch the DB).
2. Handles `OPTIONS` preflight with `Access-Control-Allow-Origin: *` and `Access-Control-Allow-Headers: authorization, content-type`.
3. Accepts POST JSON body `{ text: string, voice?: "alloy" | "echo" | "fable" | "onyx" | "nova" | "shimmer" }`. Defaults `voice` to `"nova"` (softer voice, appropriate for teen audience per research recommendation).
4. Returns `{ error: "text required" }` status 400 if `!text || text.trim().length < 1`.
5. Hard-caps input at 4000 characters: `const input = text.slice(0, 4000);` (Pitfall 2 — leaves 96-char headroom against the 4096 OpenAI limit). Long-text chunking is deferred — callers must pre-chunk.
6. POSTs to `https://api.openai.com/v1/audio/speech` with headers `Authorization: Bearer ${OPENAI_API_KEY}` and `Content-Type: application/json`. Body: `JSON.stringify({ model: "tts-1", input, voice, response_format: "mp3" })`.
7. If `!openaiRes.ok` log `console.error("openai tts error:", await openaiRes.text())` and return `{ error: "TTS failed" }` status 500 with `Content-Type: application/json`.
8. On success return the response body as a stream: `return new Response(openaiRes.body, { status: 200, headers: { "Content-Type": "audio/mpeg", "Access-Control-Allow-Origin": "*", "Cache-Control": "no-store" } })`. The `Cache-Control: no-store` is intentional — these are dynamic per-text calls; we don't want the browser caching variable audio.
9. Wraps the body parsing + fetch in a top-level `try/catch` that returns `{ error: "Internal error" }` status 500.

Add the comment `// REQUIRES: OPENAI_API_KEY in Supabase Edge Function secrets.` at the top.
</action>
