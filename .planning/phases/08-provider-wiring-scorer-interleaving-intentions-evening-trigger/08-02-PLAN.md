# Plan 08-02: Scorer Interleaving (last_shown_at de-promotion)

## Objective

Add interleaving de-promotion to `rankAssignments`. When the same class was shown at the top of the dashboard on the previous render, demote subsequent assignments from that class so the student naturally rotates between subjects — a g=0.42 effect in the spaced-interleaving literature (Rohrer & Taylor 2007; Kornell & Bjork 2008). Urgent assignments (due within 24h) bypass the penalty so deadlines still win.

## Context

The scorer in `lib/scoring/next-five-minutes.ts` already tracks `class_id` on every assignment but never uses it for cross-task decisions. The dashboard renders the top task and three runners-up; if a student does English homework, the next render typically still shows English (because English assignments tend to cluster in due dates). Interleaving asks the scorer to favor variety. Persistence is via a cookie (`diana_last_class`) — research recommends cookie-based for the App Router server-component context (no localStorage round-trip needed).

## must_haves
- `rankAssignments` signature gains a 6th parameter `lastShownClassId: string | null = null`
- When `lastShownClassId` is not null AND a scored assignment's `class_id === lastShownClassId`, subtract `INTERLEAVE_PENALTY = 15` from its score, BUT skip the penalty if its `reasons` array contains `"due now"` or `"due today"` (Pitfall 6 — urgency wins)
- Three unit tests pass: penalty applied (INTERLEAVE-01), penalty skipped for due-now (INTERLEAVE-02), no penalty when lastShownClassId is null (INTERLEAVE-03)
- `app/(app)/dashboard/actions.ts` exports `setLastShownClass(classId: string): Promise<void>` server action that calls `cookies().set('diana_last_class', classId, { httpOnly: false, sameSite: 'lax', path: '/' })`
- `app/(app)/dashboard/page.tsx` reads the cookie via `cookies().get('diana_last_class')?.value ?? null`, passes it to `rankAssignments`, and after picking `top` calls `setLastShownClass(top.class_id)` (fire-and-forget — wrap in `void` or Promise chain)

   const scored = assignments
     .filter(
       (a) =>
         a.status !== "submitted" &&
         a.status !== "graded" &&
         a.status !== "abandoned",
     )
     .map((a) => score(a, signals, now, energy, profile));

     for (const s of scored) {

         const isUrgent =
           s.reasons.includes("due now") || s.reasons.includes("due today");
         if (!isUrgent) {

**Part A — Create `app/(app)/dashboard/actions.ts`:**

```typescript
"use server";

import { cookies } from "next/headers";
import { z } from "zod";

const ClassIdInput = z.string().uuid();

/**
 * Phase 8 — Scorer interleaving. Set the cookie that drives the next render's
 * de-promotion of the same-class top task. Called fire-and-forget from the
 * dashboard server component after the top assignment is identified.
 *
 * Cookie: diana_last_class
 * - HttpOnly: false (no XSS-sensitive data; just a class id)
 * - SameSite: lax (default behavior; the cookie is only read on same-origin GET)
 * - Path: /
 * - Max-Age: 12 hours — bounded so cross-session reset is natural
 */
export async function setLastShownClass(classId: string): Promise<void> {
  const parsed = ClassIdInput.safeParse(classId);
  if (!parsed.success) return;
  const c = await cookies();
  c.set("diana_last_class", parsed.data, {
    httpOnly: false,
    sameSite: "lax",
    path: "/",
    maxAge: 12 * 60 * 60,
  });
}
```

Notes:
- Uses `await cookies()` because Next.js 15 `cookies()` returns a Promise (App Router change).
- Returns void (no value, no error response). Caller doesn't await.
- Silently no-ops on invalid UUID — defense in depth, never throws.
