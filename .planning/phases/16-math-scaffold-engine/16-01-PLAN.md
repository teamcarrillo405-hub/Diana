# Phase 16 Plan 01

## Goal

Implement the Math Scaffold Engine:
- Photo scan for math problems using GPT-4o vision.
- Socratic structured math scaffold that never gives the final answer.
- Whiteboard-style step board on assignment detail.
- Existing isomorphic worked-example flow remains available.
- Common mix-up detection, unit/dimension tracker, and graph sketch guidance.

## Acceptance Criteria

- Students can paste a problem or upload a photo from a problem-set assignment.
- Photo scan extracts problem text without solving it.
- The scaffold returns 4 to 7 student-actionable steps.
- The UI renders the steps as a board with space for student work.
- Common mix-up, unit tracker, and graph sketch sections render when present.
- Existing formula reference and worked-example tools still work.
- `math-scaffold` is deployed and ACTIVE in Supabase.
- Static gates remain green: typecheck, Vitest, tone-audit, production build.
