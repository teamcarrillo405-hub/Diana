# Phase 26 Plan 01 - PE, Health & Wellness Engine

## Scope

Deliver the v2 PE, health, and wellness layer:

- Non-judgmental activity logging.
- Student-owned personal goal setting without body metric fields.
- Health class scaffold for reproductive health, nutrition, mental health, and course questions.
- CPR / first aid starter flashcards using the existing FSRS deck.
- Sleep + recovery logging connected to next-day task sizing.

## Implementation Notes

- Migration `0027_health_wellness_engine.sql` adds `wellness_activity_logs`, `wellness_goals`, and `sleep_logs` with owner RLS.
- The migration also extends `task_signals.kind` for `overwhelmed`, `mood_checkin`, `activity_log`, and `sleep_log`.
- `/wellness` is the student surface for activity logs, personal goals, sleep logs, and CPR / first aid card creation.
- `lib/wellness/health.ts` keeps goal validation, sleep-to-energy adjustment, CPR / first aid cards, and health scaffold parsing deterministic/testable.
- Dashboard reads the latest sleep log and lowers the default energy only when the student has not chosen a different energy.
- `HealthHelper` appears on PE/health assignment detail pages and wraps the new `health-scaffold` Edge Function.

## Acceptance Mapping

- F116 Activity log: `wellness_activity_logs`, `/wellness` activity form.
- F117 Personal goal setting: `wellness_goals`, `goalTextIsAllowed`, `/wellness` goal form.
