# Phase 33 Plan - Personalization Settings v2

## Goal

Deliver F162-F168: per-class AI verbosity, granular notifications, student-owned data export, account deletion request flow, multi-device handoff, encrypted profile backup, and privacy dashboard.

## Scope

- Add durable profile preference columns for per-subject AI style, notification preferences, and privacy preferences.
- Add `session_handoffs` for "continue where you left off" state across devices.
- Add `data_deletion_requests` for COPPA deletion audit and immediate AI disablement.
- Replace `/export` with a live data and privacy dashboard.
- Export user-owned data as JSON and a lightweight PDF inventory.
- Export/import an encrypted profile backup through browser Web Crypto.
- Add privacy inventory, AI-context visibility, and category-level data clearing.
- Add an authenticated route tracker that upserts current app route into `session_handoffs`.
- Expose data/privacy controls from Settings and mark F19 live in the feature registry.

## Acceptance Mapping

- F162 Expanded settings: `/export` supports per-class AI style selection backed by `profiles.ai_verbosity_by_subject`.
- F163 Notification preferences: `/export` supports granular notification toggles backed by `profiles.notification_preferences`.
- F164 Data export: JSON export includes profile, classes, assignments, notes, flashcards, AI interactions, mastery, and share links; PDF export includes an inventory summary.
- F165 Account deletion: request flow disables AI immediately and records a `data_deletion_requests` row with export offered.
- F166 Multi-device handoff: authenticated app layout tracks current route and `/export` shows the latest handoff.
- F167 Profile backup: browser-encrypted backup exports/imports profile preferences.
- F168 Privacy dashboard: `/export` shows data inventory, AI context, and category-level clearing controls.
