# Phase 29 Plan 01 - Vocabulary + Reading Scaffold Engine

## Scope

Deliver the v2 reading and vocabulary layer:

- Hover vocabulary with context-aware definitions.
- Reading-level adaptation for notes and assignment reading text.
- Annotation mode for highlighted note/assignment text.
- Read-aloud continuity through the existing synchronized TTS highlight mode.
- Vocabulary builder that saves hover words into the FSRS flashcard system.
- Phonics support with syllable chunks, stress, and pronunciation.
- Context-clue trainer before opening the definition on first encounter.

## Implementation Notes

- Migration `0029_vocabulary_reading_scaffold.sql` adds `vocabulary_terms` and `reading_annotations` with owner RLS.
- `vocab-hover` now returns structured JSON: definition, context clue, and phonics support.
- `reading-level` is a new Edge Function for simpler / more-detail text adaptation.
- `AccessibleReadingText` marks word spans with `data-vocab-word` so hover support works across existing note, transcript, assignment, and reading-panel text.
- `VocabHoverProvider` opens a local context-clue/phonics popover first, then fetches definitions on request or after the word has been seen once.
- `ReadingLevelAdapter`, `ReadingAnnotationControl`, and `AssignmentReadingBlock` are reusable client controls backed by server actions.
- Vocabulary cards reuse the existing `flashcards` FSRS table rather than creating a separate review engine.

## Acceptance Mapping

- F135 Hover vocabulary: word spans open `VocabHoverProvider`, which calls `vocab-hover` with owner, class AI mode, word, and context.
- F136 Reading level slider: `ReadingLevelAdapter` exposes Original / Simpler / More detail and invokes `reading-level`.
- F137 Annotation mode: `ReadingAnnotationControl` saves highlighted text + note rows into `reading_annotations`.
- F138 Read-aloud mode: existing `TtsHighlightButton` remains wired on notes, transcripts, assignment headers, and reading panels.
- F139 Vocabulary builder: `saveVocabularyCard` creates an FSRS flashcard and a `vocabulary_terms` row from hover definitions.
- F140 Phonics scaffold: `phonicsBreakdown` and `vocab-hover` phonics JSON provide syllables, stress, and pronunciation.
- F141 Context clue trainer: first hover shows context clue and phonics before the definition request.
