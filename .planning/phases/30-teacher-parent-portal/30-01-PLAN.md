# Phase 30 Plan 01 - Teacher + Parent Portal

## Scope

Deliver the student-controlled teacher and parent portal layer:

- Teacher assignment creation with rubric text and assignment-level AI policy.
- Class roster / invite management.
- Assignment analytics by class without individual comparison or ranking.
- Parent dashboard with read-only weekly summary.
- Teacher progress notes visible to student and optionally parent-visible.
- Accommodation confirmation against the active student profile.
- IEP / 504 import continuity from the existing Settings import flow.

## Implementation Notes

- Migration `0030_teacher_parent_portal.sql` adds `assignments.ai_mode_override`, `class_roster_members`, `teacher_progress_notes`, and `accommodation_confirmations`.
- `/teacher-share` is now an authenticated portal instead of a placeholder.
- `/parent-share` is now a read-only dashboard preview plus existing share-link management.
- Public `/share/[token]` parent summaries include only `visible_to_parent` progress notes.
- Assignment detail computes effective AI policy from class mode plus assignment override.
- `lib/portal/teacher.ts` keeps AI policy and class analytics deterministic/testable.

## Acceptance Mapping

- F142 Teacher portal: `/teacher-share` assignment creation form creates assignments with class, due date, rubric text, and AI policy.
- F143 Class management: `class_roster_members` and portal roster form store invited teachers/students/aides/guardians.
- F144 Assignment analytics: `classCompletionAnalytics` shows per-class completion rate without individual names.
- F145 Parent dashboard: `/parent-share` and public parent summaries show read-only effort/upcoming-work data.
- F146 Progress notes: `teacher_progress_notes` stores teacher-authored notes, with parent visibility flag.
- F147 Accommodation confirmation: `accommodation_confirmations` stores a snapshot of active accommodations.
- F148 IEP integration: teacher portal embeds the existing IEP / 504 import flow that auto-configures profile settings.
