# Phase 35 Plan - v2.0 Hardening + Launch

## Goal

Complete the v2.0 hardening and launch readiness layer: accessibility/security/performance audit artifacts, load/mobile/beta plans, COPPA retention enforcement, critical-path coverage review, and launch documentation.

## Scope

- Add COPPA retention enforcement migration and service-role purge function.
- Add launch-readiness helper module and tests.
- Add `npm run launch-audit` gate for required docs, critical-path test coverage, and performance budget constants.
- Add launch documentation for accessibility, performance budgets, security, retention, mobile device matrix, and beta rollout.
- Keep the final hardening evidence tied to existing green static gates and linked migration parity.

## Acceptance Mapping

- Accessibility audit: `docs/launch/LAUNCH_READINESS.md` captures keyboard, screen reader, contrast, reduced motion, and reading-support checks.
- Security audit: `docs/launch/SECURITY_AND_RETENTION.md` maps OWASP areas to Diana controls.
- Performance budget enforcement: `lib/launch/readiness.ts` and `npm run launch-audit` lock LCP/FID/CLS budgets.
- Load testing: `docs/launch/MOBILE_AND_BETA_MATRIX.md` includes beta/load-readiness exit criteria; launch gate keeps production readiness explicit.
- COPPA data retention: migration `0034_launch_hardening_retention.sql` adds `purge_due_deletion_requests`.
- Coverage review: launch audit requires at least 80 percent critical-path test coverage.
- Mobile matrix and beta rollout: documented in `docs/launch/MOBILE_AND_BETA_MATRIX.md`.
- Launch documentation: docs are required by `npm run launch-audit`.
