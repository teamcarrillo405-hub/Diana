# Phase 31 Plan - Platform Intelligence + Analytics

## Goal

Deliver F149-F154: internal analytics, AI token cost monitoring, feature flags, app monitoring, web vital budget tracking, and UI-only experiment controls.

## Scope

- Add student-owned Supabase tables for feature flags, analytics events, app monitor reports, performance events, and UI experiments.
- Replace `/insights` with a live operational dashboard.
- Add feature flag and experiment upsert actions.
- Add authenticated monitoring API routes for usage events, app reports, and web vitals.
- Add a lightweight authenticated-layout telemetry component for page views, route duration, and Core Web Vitals.
- Keep diagnosis context anonymized into categories only.
- Enforce UI-only experiment surfaces; no content, AI, privacy, safety, IEP, or accommodation experiments.

## Acceptance Mapping

- F149 Internal analytics: route/page usage events, route session duration, assignment completion rate.
- F150 AI cost dashboard: daily token use by AI feature for the signed-in account.
- F151 Feature flags: owner-scoped flags can be toggled and rolled out without deploy.
- F152 App monitoring: authenticated report endpoint stores anonymized diagnosis category tags.
- F153 Performance monitoring: Core Web Vitals are stored per route with budget status.
- F154 A/B testing: UI experiment table/actions reject protected surfaces.
