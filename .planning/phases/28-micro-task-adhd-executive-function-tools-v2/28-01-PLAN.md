# Phase 28 Plan 01 - Micro-Task + ADHD Executive Function Tools v2

## Scope

Deliver the v2 executive-function layer:

- Make every AI task breakdown produce five-minute-or-smaller micro-steps.
- Add a three-second task-initiation ritual before focus sessions start.
- Upgrade body-doubling with a study-type breakdown.
- Adapt Pomodoro breaks based on session mood and task difficulty.
- Restore interrupt-recovery context with exact step and scroll position.
- Add global quick capture for working-memory offload.
- Show a task-switching cost cue when moving between assignments.
- Preserve time-estimate calibration through the existing assignment time-log and type-estimate substrate.

## Implementation Notes

- No new database migration was needed; Phase 28 reuses `assignment_steps`, `inbox_items`, `assignment_time_log`, `assignment_type_estimates`, and local browser recovery state.
- `task-breakdown` now asks for and validates steps at `<=5` minutes through `MAX_MINUTES_PER_STEP`.
- `lib/executive/session.ts` keeps adaptive break, body-double category, and task-switch cue logic deterministic/testable.
- `/timer` accepts mood and difficulty context and starts through a visible `Ready.` countdown.
- Assignment task breakdown stores local recovery context under `diana:assignment-recovery:{assignmentId}`.
- Global `QuickCapture` inserts unclassified `inbox_items` without leaving the current task.

## Acceptance Mapping

- F127 Make it smaller: `MAX_MINUTES_PER_STEP`, parser clamp, and `task-breakdown` prompt enforce five-minute steps.
- F128 Task initiation ritual: `TimerUi` shows a three-second `Ready.` countdown before `start`.
- F129 Body-doubling v2: `/body-double` renders Math, Reading, Writing, and Review category counts.
- F130 Pomodoro++: `adaptiveBreakMinutes` adjusts break length from mood, difficulty, and work length.
- F131 Interrupt recovery v2: assignment breakdown stores step index and scroll position and restores on return.
- F132 Working memory offload: authenticated app layout renders global `QuickCapture`.
- F133 Task switching cost display: assignment detail shows the 15-minute warm-up cue when context changes.
- F134 Time estimate calibration: existing `assignment_time_log`, `assignment_type_estimates`, and new-assignment calibration hints remain the durable calibration path.
