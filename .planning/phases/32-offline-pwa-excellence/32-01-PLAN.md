# Phase 32 Plan - Offline + PWA Excellence

## Goal

Deliver F155-F161: offline note editing, offline flashcard review, offline assignment status updates, background sync, install experience, optional reminders, and PWA share target.

## Scope

- Add a first-party service worker for core route caching, navigation fallback, push notifications, and background sync messages.
- Add an IndexedDB-backed offline queue for note saves, assignment status changes, flashcard review ratings, and cached due flashcard snapshots.
- Register the service worker from the authenticated app layout.
- Drain queued offline work when the app comes online or receives the service worker sync message.
- Add Settings controls for install prompt, queue sync, and optional calm reminders.
- Add a reminder API for the service worker notification path.
- Add a PWA share target route that creates a note from shared text, URL, photo, PDF, or text file.
- Add an offline fallback route and manifest share target/shortcut/icon metadata.

## Acceptance Mapping

- F155 Full offline note editing: note saves queue locally and drain to `createNote` / `saveNote`.
- F156 Offline flashcard review: due-card snapshots cache locally and ratings queue for sync.
- F157 Offline assignment viewing/status: service worker caches visited routes and status transitions queue locally.
- F158 Background sync: `diana-offline-sync` posts a drain message to authenticated clients.
- F159 Install prompt optimization: Settings exposes install prompt state and offline queue status.
- F160 Push notifications: optional local reminder notifications use the service worker and calm copy.
- F161 iOS PWA share target: manifest posts shared files/text to `/api/share-target`, which creates a note.
