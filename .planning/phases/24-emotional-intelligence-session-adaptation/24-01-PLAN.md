# Phase 24 Plan 01 - Emotional Intelligence + Session Adaptation

## Scope

Deliver the v2 emotional intelligence layer:

- Daily mood check-in with permanent dismissal.
- Session adaptation for `meh` and `rough` moods.
- Frustration support after repeated low-confidence attempts.
- Global "I'm overwhelmed" escape valve that creates a 5-minute child task when used on an assignment.
- Weekly reflection prompt with AI mirror.
- Quiet milestone acknowledgement without confetti, ranking, or public celebration.
- Burnout/reset cue based on time logs, rough mood, and overwhelmed signals.

## Implementation Notes

- Migration `0025_emotional_intelligence_session_adaptation.sql` adds mood cadence fields to `profiles` and `student_reflections`.
- `lib/emotional/session.ts` keeps adaptation, reflection timing, frustration threshold, reset cues, and quiet milestones deterministic/testable.
- The dashboard is the session-start surface for mood check-in, adaptation card, weekly reflection, and reset cue.
- The authenticated app layout renders the overwhelmed button across app/task surfaces.
- Flashcard review uses "Again" ratings as the first concrete repeated-attempt signal for F105.
- `weekly-reflection` is a new Edge Function using Claude Haiku 4.5, safety prompts, budget checks, and `ai_interactions` logging.

## Acceptance Mapping

- F103 Session mood check-in: `MoodCheckIn`, profile cadence fields, permanent disable.
- F104 Session adaptation: `sessionAdaptationForMood`, dashboard rough/light mode, rough timer defaults.
- F105 Frustration detection: `shouldOfferDifferentApproach`, flashcard review threshold prompt.

- F107 Weekly reflection: `WeeklyReflection`, `student_reflections`, `weekly-reflection` Edge Function.
